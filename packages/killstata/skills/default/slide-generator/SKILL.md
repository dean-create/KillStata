# Slide Generator

Use this after a paper draft or baseline outputs already exist.

## Tool path

1. Resolve the `research_brief` and baseline outputs.
2. Optionally resolve the paper draft and heterogeneity bundle.
3. Call `slide_generator`.
4. Read `slides.json` and `speaker_notes.md` before presenting the deck.

## Rules

- One slide should carry one main claim.
- Prefer existing tables or figures when possible.
- If no chart exists, recommend a visual instead of inventing one.
