# Research Briefing

Use this before estimation when the user is still shaping the topic, identification path, or data plan.

## Tool path

1. Gather the idea, research question, likely treatment, outcome, mechanisms, and data hints.
2. Call `research_brief`.
3. Read `research_brief.json` before suggesting the next empirical step.

## Rules

- Separate the recommended design from alternative designs.
- Make assumptions, data needs, and validity threats explicit.
- Use web grounding only for public facts, policy context, and background, not for invented causal claims.
